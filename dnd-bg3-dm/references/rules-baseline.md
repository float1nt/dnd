# Rules Baseline

Use this as the default adjudication layer for D&D 5e-inspired play.

## Core Loop

- State the situation.
- Ask for player intent.
- Call for a check only when failure changes the scene.
- Resolve with advantage, disadvantage, or a DC.
- Apply consequences immediately.
- Update campaign state.

## Ability Checks

- Use one ability plus one relevant skill when appropriate.
- Default DC bands:
  - 5 easy
  - 10 routine
  - 15 hard
  - 20 very hard
  - 25 extreme
- If the task is impossible, say so instead of rolling.
- If multiple approaches work, let the player choose the approach.

## Combat

- Run initiative in descending order.
- On a turn, track movement, action, bonus action, reaction, and free interaction.
- Apply cover, range, conditions, concentration, and line of sight as needed.
- Keep monster abilities short and concrete in play notes.

## Rests and Recovery

- Use short rests and long rests as the main pacing tools.
- Track hit dice, spell slots, and expendable features explicitly.
- Do not restore resources unless a rule or rest says so.

## Spells

- Track prepared, known, and granted spells separately.
- Record spell slots by level.
- If the exact spell text matters and is not in the campaign notes, ask the user or use a rules reference instead of inventing the effect.
- Never guess a spell duration, range, or target if it affects the current scene.

## Treasure and Progression

- Log all loot, gold, consumables, and attunement state.
- Use milestones or XP only as chosen in the campaign record.
- Record level-ups in the active campaign state immediately.

## House Rule Discipline

- Mark any BG3-style change as a house rule.
- Keep house rules local to the active campaign unless the user wants them shared.
- When a rule is uncertain, say `Unknown` and pause for clarification.
